# PermissionsExPlus

PermissionsExPlus is a maintained fork of [PermissionsEx](https://github.com/PEXPlugins/PermissionsEx) (last upstream release **1.23.4**). Upgrade path: **PermissionsEx 1.23.4 → PermissionsExPlus 3.0.0-SNAPSHOT**. See the [migration guide](https://permissionsexplus.rono.dev/faq/migrate-from-v1) on the docs site.

**Documentation website:** [permissionsexplus.rono.dev](https://permissionsexplus.rono.dev/)

## Documentation site development

All site tooling lives under `website/`:

```bash
cd website
npm install
npm start
```

Open [http://localhost:3000](http://localhost:3000) to preview changes.

Site variables (`version`, `repo`, `siteUrl`) live in `website/site-vars.json`, generated by:

```bash
cd website && npm run sync-site-vars
```

When `pom.xml` is present, the script reads `${project.version}` from Maven automatically.

Production build:

```bash
cd website
npm run build:javadoc   # optional locally; CI builds Javadoc before deploy
npm run build
npm run serve
```

The static site is written to `website/build/`. Pushes to `main` that touch `website/**` trigger [`.github/workflows/deploy-docs.yml`](.github/workflows/deploy-docs.yml).

| Path | Purpose |
|------|---------|
| `website/docs/` | Documentation pages (Markdown / MDX) |
| `website/src/` | React components and theme CSS |
| `website/static/` | Static assets (images, CNAME; Javadoc generated at build time) |
| `examples/` | Sample `config.yml` and `permissions.yml` |
| `website/docusaurus.config.ts` | Site configuration |

## Build from source

```bash
mvn clean package
```

Universal jar:

```bash
mvn clean package -pl bootstrap -am
```

Output: `bootstrap/target/PermissionsExPlus-{version}.jar`
